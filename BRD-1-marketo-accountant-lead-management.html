1️⃣ Executive Summary

This BRD defines the technical requirements to integrate accountant personas into the Marketo lead management pipeline. The accountant channel is forecast to deliver 3,471 IES contracts in 2H (a 24x scale-up from 1H's 141 contracts). Without automation of accountant ingestion, enrichment, scoring, and routing in Marketo, this forecast is at serious risk. Estimated Business Impact: 350–400 incremental IES contracts (~$6.65M–$7.6M revenue).

2️⃣ Context & Current State

Current State

  • Marketo does not ingest, identify, or score accountant leads.
  • Accountant leads arriving via IPD abandonment, webinars, or content engagement enter a "black hole" — not scored, not routed, not actionable by Sales.
  • Manual enrichment processes ("fire drills") are the only mechanism, cannot scale.
  • L2A matching logic maps all leads to Client Accounts; accountants should map to Firm Account (employer).

Future State

  • Marketo ingests accountant leads from all upstream sources (IPD, webinars, content engagement, ABM handoffs).
  • Automated enrichment via 1P BPP pipeline identifies accountant persona type and firm type at ingestion.
  • L2A logic correctly resolves accountant leads to Firm Accounts in SFDC.
  • Scoring model evaluates accountant-specific engagement signals, triggers lead pass to Sales at threshold.
  • Nurture campaigns at scale with proper segmentation by firm tier and engagement stage.

3️⃣ Problem Statements

Problem 1 — Sales Seller perspective

  • • I am a Mid-Market Sales Seller
  • • I am trying to contact accountant leads during the golden window when they show intent
  • • But I don't receive accountant leads in a timely or prioritized manner
  • • Because Marketo does not ingest, score, or route accountant leads — they require manual enrichment "fire drills"
  • • Which makes me feel like I'm missing high-intent opportunities and can't hit my 2 IES contracts/week target

Problem 2 — Marketing Ops perspective

  • • I am a Marketing Operations Manager
  • • I am trying to run automated nurture campaigns for accountant leads at scale
  • • But accountant leads are not identified or classified in Marketo
  • • Because there is no accountant identification field, no enrichment pipeline, and no L2A logic for accountant-to-firm mapping
  • • Which makes me feel like I'm running manual processes that can't support the 24x scale-up planned for 2H

Problem 3 — GTM Tech perspective

  • • I am a GTM Tech Engineer
  • • I am trying to ensure accountant leads flow through the lead management pipeline correctly
  • • But the current L2A logic maps accountants to Client Accounts instead of their employing Firm Account
  • • Because the system was built for direct customer leads and has no conditional routing logic for accountant personas
  • • Which makes me feel like the data architecture is fundamentally misaligned with the accountant GTM strategy

4️⃣ Hypothesis

If we integrate accountant identification, enrichment, scoring, and L2A routing into Marketo

then Sales will receive prioritized, correctly-routed accountant MQLs during the golden window, enabling timely outreach and nurture at scale

as measured by:

  1. MQL volume for accountant leads
  2. MQL-to-meeting conversion rate
  3. Time from lead creation to Sales contact (golden window SLA)
  4. Incremental IES contracts attributable to Marketo-sourced accountant leads

Supplemental: Technical Appendix

Technical Requirements Detail

Schema Changes

Requirement ID Description
MKT-SCH-001Add is_accountant (Boolean) to Person object. Default false; set by enrichment.
MKT-SCH-002Add accountant_persona_type (Picklist: Partner, Staff, Decision-Maker, Unknown) to Person. Populated by BPP.
MKT-SCH-003Add firm_tier (Picklist: National, HVAM, Unmanaged) to Person. Derived from SFDC.
MKT-SCH-004Add firm_account_id__c (String 18-char SFDC ID) to Person. FK to Firm Account.

Enrichment Pipeline

Requirement ID Description
MKT-ENR-001On new lead creation, invoke BPP enrichment API to classify accountant persona, firm type, firm tier.
MKT-ENR-002Map BPP response → Marketo fields.
MKT-ENR-00360s SLA, max 3 retries, exponential backoff.
MKT-ENR-004On failure, flag enrichment_status='failed', route to manual queue, alert GTM Tech via Slack.

L2A Logic

Requirement ID Description
MKT-L2A-001If is_accountant=true, match to Firm Account using firm name + domain fuzzy match. Else use existing logic.
MKT-L2A-002Weighted matching: exact domain (1.0), firm name Levenshtein <3 (0.7), BPP firm ID (1.0). Highest confidence wins.
MKT-L2A-003If no match >0.7 confidence, route to "Unmatched Accountant" queue.
MKT-L2A-004After resolution, stamp lead_owner with Firm Account's SFDC owner.

Scoring Model

Requirement ID Description
MKT-SCR-001Behavioral weights: webinar reg (+5), attendance (+15), IPD start (+10), IPD abandon (+8), email open (+2), email click (+5), content download (+10), form fill (+12), hand-raiser (+25).
MKT-SCR-002Demographic weights: National (+20), HVAM (+15), Unmanaged (+5). Decision-Maker (+15), Partner (+10), Staff (+5).
MKT-SCR-003Score decay: -10% every 30 days of inactivity.
MKT-SCR-004MQL threshold ≥100 points (configurable per firm tier).
MKT-SCR-005Sync marketo_lead_score and score_breakdown (JSON) to SFDC on every change.

SFDC Sync

Requirement ID Description
MKT-SYN-001On MQL, change status, set mql_date, create SFDC Task for account owner.
MKT-SYN-002SFDC opportunity stage changes sync back to Marketo for lifecycle/nurture.
MKT-SYN-003Marketo→SFDC within 5 min. SFDC→Marketo within 15 min.

Nurture Architecture

Requirement ID Description
MKT-NUR-0016 streams (Webinar Attended, Webinar Not-Attended, Hand-Raisers, Post-Meeting Non-Buyers, Stalled/Cold, IPD Abandoners).
MKT-NUR-002Entry via Smart List criteria on is_accountant, persona_type, firm_tier, activity, lifecycle.
MKT-NUR-003Cadence 1 email/7 days. Suppress on open opp ≥ Meeting Scheduled.
MKT-NUR-004Exit on MQL, Closed Won, Unsubscribe, Hard Bounce, or Sales opt-out.

Technical Recommendations (from the TPM)

  • Define canonical Accountant data model in schema registry. Add enrichment_timestamp/enrichment_source fields.
  • Specify BPP API as versioned OpenAPI spec. Implement DLQ for failed enrichments.
  • Parameterize scoring weights in config layer. Add score audit log. Backtest model against 1H data.
  • Document L2A matching algorithm formally. Track L2A match rate as KPI (target >90%).
  • Define SFDC field mapping table explicitly. Implement sync monitoring (p50/p95/p99 latency).
  • Build lead funnel dashboard. Define SLIs/SLOs for pipeline.
  • Require UAT per phase. Provide sample test data with expected outcomes.
